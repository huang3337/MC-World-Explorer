# DECISION-002：Region 首轮采用标准 Anvil 与可扩展分层设计

- **状态**：已确认
- **确认日期**：2026-07-14
- **确认人**：项目开发者
- **适用版本**：V0.2 及后续 Region 兼容扩展

## 决策背景

V0.2 需要只读解析主世界 `region/*.mca`，为后续地表采样和缩略图生成提供区块 NBT 数据。不同 Minecraft 版本、服务端配置和 Mod 环境可能使用不同压缩方式、外部区块文件或区块 NBT 结构，因此首轮实现既不能无边界覆盖所有历史格式，也不能把当前支持范围写死在单一读取流程中。

本决策只确定 Region 容器读取和解压阶段的首轮兼容范围。现代 Section、Palette、旧版区块结构和 Mod 方块数据属于后续阶段。

## 备选方案

### 方案 A：仅实现标准 Anvil 基础路径

首轮支持标准 `.mca`、GZIP、ZLIB 和未压缩区块，其他格式直接视为不支持。

优点是实现简单、风险较低；缺点是如果读取、解压和区块 NBT 结构耦合在一起，后续增加格式时可能需要重写已有逻辑。

### 方案 B：首轮同时实现 LZ4 和外部区块

在方案 A 基础上增加 LZ4 解压和外部 `.mcc` 区块读取。

优点是首轮兼容范围更广；缺点是需要新增依赖和更多文件边界处理，扩大当前单一阶段的实现与测试范围。

### 方案 C：首轮覆盖旧版和特殊 Mod 格式

同时处理旧 `.mcr`、多代区块 NBT 结构以及特殊 Mod 自定义格式。

优点是目标范围最大；缺点是容器、压缩、数据版本和 Mod 差异会同时进入首轮开发，难以保持简单和验证只读安全性。

## 最终选择

采用 A+：保持方案 A 的首轮实现范围，同时使用轻量分层为后续扩展保留明确位置。

首轮实际支持：

- 标准 Java 版 Anvil `.mca` 文件。
- Region 位置表和时间戳表。
- 区块扇区定位、长度读取和边界验证。
- 压缩标识 `1`（GZIP）、`2`（ZLIB）和 `3`（未压缩）。
- 解压后的区块 NBT 字节输出。

首轮只识别并明确报告、但不实际读取：

- 压缩标识 `4`（LZ4）。
- 压缩字节带外部区块标志的 `.mcc` 数据。
- 旧 `.mcr` 文件。
- 特殊 Mod 自定义 Region 容器或未知压缩标识。

## 分层约束

- Region 容器负责文件头、区块位置、时间戳、长度和只读字节读取。
- 压缩层根据压缩标识解压区块负载，不解释 NBT 内容。
- 区块 NBT 层在后续阶段根据 `DataVersion` 和数据结构解析 Section、Palette 与方块状态。
- Region 容器不得依赖 JavaFX、缓存、图片渲染或存档扫描 UI。
- 不建立公开插件系统、反射加载或 `ServiceLoader` 扩展框架；当前仅保留内部接口和枚举边界。

## 只读与失败约束

- 仅以读取模式打开 Minecraft 文件，不提供写回、修复、截断、移动或删除接口。
- 缺失区块与损坏区块必须区分；缺失是正常结果，损坏和不支持格式返回明确原因。
- 所有偏移、扇区数、区块长度和解压后大小都必须检查边界。
- 单个区块解压后最多允许 64 MiB，超过限制时终止该区块读取，防止损坏数据导致无限内存占用。
- 一个区块失败不得导致修改 Region 文件，也不得阻止调用方继续处理其他区块。

## 扩展方式

- 支持 LZ4 时增加对应解压实现和必要依赖，不修改 Region 位置解析。
- 支持外部区块时增加外部负载解析器，并继续复用压缩层和 NBT 层。
- 支持旧 `.mcr` 时增加独立容器读取器，不在标准 `.mca` 读取器中堆叠版本分支。
- 支持旧版或 Mod 区块数据时在 NBT 数据层按 `DataVersion` 或结构特征扩展，不修改容器和解压层。

## 对 V0.2 的影响

- 当前阶段只实现 Region 文件头、区块定位和三种基础压缩的解压。
- 不在本阶段实现 Section、Palette、地表采样、颜色映射、PNG、缓存或 UI。
- 合成测试必须覆盖正常数据、缺失区块、负坐标映射、截断文件、非法偏移、非法长度、未知压缩、LZ4 标识和外部区块标志。
- 真实存档验收必须在读取前后比较 Region 文件大小、修改时间和 SHA-256，证明没有写入。
- 扩大首轮实现范围或新增第三方压缩依赖前，必须更新决策并由开发者再次确认。

## 长期影响

该选择优先保证 V0.2 能以可验证的最小范围读取主流标准 Anvil 数据，同时避免把未来兼容性建立在一次性代码上。后续扩展通过新增压缩实现、负载来源或 NBT 数据解析器完成，而不是推翻已经测试的 Region 定位逻辑。
