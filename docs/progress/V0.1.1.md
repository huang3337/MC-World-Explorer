## 开发内容

### 一、引入日志框架 + 全局异常捕获

**解决的问题**：ISSUE-APP-001, ISSUE-APP-002, ISSUE-LEVELDAT-003, ISSUE-SCANNER-003, ISSUE-LISTCELL-002, ISSUE-TREECELL-001

**目标**：让所有错误信息写入日志文件，程序不再静默吞掉异常

**修改文件**：
- `build.gradle` — 添加 Logback 依赖
- `App.java` — 添加全局异常处理器，替换 System.err.println
- `LevelDatReader.java` — 替换 System.err.println 为 logger
- `WorldScanner.java` — 替换 System.err.println 为 logger
- `WorldListCell.java` — 替换静默 catch 为 logger
- `WorldTreeCell.java` — 替换静默 catch 为 logger

**新增文件**：
- `src/main/resources/logback.xml` — 日志配置（输出到文件 + 控制台）

**验收标准**：
- 程序运行后在 `%APPDATA%/MC World Explorer/logs/` 生成日志文件
- 所有原 System.err.println 位置改为 logger 调用
- 未捕获异常被记录到日志文件

---

### 二、统一错误处理策略

**解决的问题**：ISSUE-LEVELDAT-002, ISSUE-LEVELDAT-005, ISSUE-SCANNER-004, ISSUE-SCANNER-005, ISSUE-CONTROLLER-003

**依赖**：需在"引入日志框架"之后进行

**目标**：LevelDatReader 和 WorldScanner 的错误处理方式一致，降级对象不再混入正常列表

**修改文件**：
- `WorldInfo.java` — 添加 `boolean parsed` 字段
- `LevelDatReader.java` — 解析失败时设置 `parsed = false`，删除硬编码字符串 `"解析失败"`
- `WorldScanner.java` — 只将 `parsed == true` 的对象加入列表
- `MainController.java` — 用 `info.isParsed()` 替代字符串判断

**验收标准**：
- 解析失败的世界不再出现在存档列表中
- `LevelDatReader` 的三种失败情况统一用同一种方式处理
- `MainController` 不再依赖硬编码字符串 `"解析失败"`

---

### 三、提取 GameType 枚举

**解决的问题**：ISSUE-WORLDINFO-001, ISSUE-CONTROLLER-002

**目标**：将游戏模式的 int 映射提取为类型安全的枚举

**修改文件**：
- `WorldInfo.java` — `gameType` 字段类型从 `int` 改为 `GameType` 枚举
- `LevelDatReader.java` — 用 `GameType.fromId()` 替代直接赋值
- `MainController.java` — 删除 switch，用 `gameType.getDisplayName()` 替代

**新增文件**：
- `src/main/java/com/mcworldexplorer/world/GameType.java` — 枚举定义

**验收标准**：
- 游戏模式显示与之前完全一致（Survival/Creative/Adventure/Spectator）
- MainController 中不再有 gameType 的 switch 语句
- 无效的 gameType 值有默认处理（回退到 Survival）

---

### 四、WorldInfo 数据验证 + null 防御

**解决的问题**：ISSUE-WORLDINFO-002, ISSUE-WORLDINFO-003, ISSUE-WORLDINFO-005

**目标**：WorldInfo 的 setter 保证不会存入无效数据，切断 null 传播链

**修改文件**：
- `WorldInfo.java` — 关键 setter 添加验证逻辑，`toString()` 补充遗漏字段

**验收标准**：
- `setLevelName(null)` 不会导致 levelName 为 null
- `setGameType(999)` 会被修正为有效值
- `toString()` 输出包含 folderPath、lastPlayed、randomSeed、playerPos

---

### 五、清理死代码 + 修复图片加载

**解决的问题**：ISSUE-LISTCELL-001, ISSUE-LISTCELL-003, ISSUE-TREECELL-002, ISSUE-TREECELL-004, ISSUE-FXML-001

**目标**：删除未使用的代码，修复图片路径构造方式，优化图标加载性能

**修改文件**：
- `WorldListCell.java` — 删除整个文件
- `main.fxml` — 删除未使用的 `ListView` 导入
- `WorldTreeCell.java` — 图标加载添加缓存

**验收标准**：
- 项目编译通过，功能不受影响
- WorldTreeCell 的图标加载不再每次都重新解析路径
- 不存在未使用的导入

---

### 六、代码规范整理

**解决的问题**：ISSUE-APP-003, ISSUE-CONTROLLER-004, ISSUE-CONTROLLER-005, ISSUE-CONTROLLER-006, ISSUE-CONTROLLER-007, ISSUE-WORLDINFO-004, ISSUE-FXML-002, ISSUE-FXML-003, ISSUE-FXML-004, ISSUE-TREECELL-003

**依赖**：需在"提取 GameType 枚举"之后进行

**目标**：消除魔法数字、硬编码字符串，提升代码可读性

**修改文件**：
- `App.java` — 窗口尺寸提取为常量
- `main.fxml` — 删除重复的 prefHeight/prefWidth，提取内联样式
- `WorldTreeCell.java` — CSS 样式移到外部文件
- `MainController.java` — SimpleDateFormat 提取为常量，seed=0 判断修复，玩家坐标添加"无数据"状态，UI 字符串提取为常量
- `WorldInfo.java` — 多变量声明改为单行

**新增文件**：
- `src/main/resources/css/styles.css` — 统一样式表

**验收标准**：
- 窗口尺寸只在一处定义
- CSS 样式集中在 styles.css 中
- seed=0 不再误显示为 "Unknown"
- 无玩家数据时显示 "Unknown" 而非 "(0.0, 0.0, 0.0)"
