## 开发目标

在 V0.2 主世界地表缩略图基础上引入统一维度图层系统，使主世界、下界、末地和真实样本中可验证的 Mod 维度共用同一套预览请求、采样、渲染与缓存流程，并提供地表总览和 32 格洞穴高度带。

## 已确认边界

- `SURFACE_OVERVIEW` 在区块现有垂直范围内寻找每个 X/Z 柱的最高非空气方块。
- `HEIGHT_BAND` 按 32 格划分高度，在范围内寻找上方至少保留两格空气的最高地面。
- 玩家当前位于目标维度且坐标可靠时，默认使用玩家 Y 所在高度带。
- 主世界、末地和未知 Mod 维度缺少目标维度坐标时默认使用地表总览。
- 下界缺少目标维度坐标时默认使用包含 `Y=64` 的高度带。
- 图层按需生成，缓存按世界、维度、中心、图层类型、高度范围和渲染版本隔离。
- 所有 Minecraft 数据只读；缓存继续写入程序根目录 `cache/`。

重大决策见 `docs/decisions/DECISION-003.md`，实现设计见 `docs/superpowers/specs/2026-07-18-unified-dimension-layer-system-design.md`。

## 实际完成

### 1. 维度发现与元数据

- 新增统一维度描述，区分主世界、下界、末地和 Mod 维度。
- 支持主世界 `region/`、下界 `DIM-1/region`、末地 `DIM1/region`。
- 结合 `D:\MC\.minecraft\versions` 的真实整合包样本，支持发现 `dimensions/<namespace>/<dimension path>/region`。
- 维度发现只遍历目录结构，进入 `region` 后立即停止向下扫描，避免枚举大型 Region 文件树。
- `level.dat` 读取新增 `Data.Player.Dimension`，兼容字符串维度 ID 和旧数字维度编号。

### 2. 统一图层与请求模型

- 新增 `PreviewLayerType`、`PreviewLayer`、`DimensionHeightRange` 和 `PreviewRequest`。
- 主世界固定采用 `-64..319`，下界和末地采用 `0..255`；Mod 维度从真实可解析区块 Section 范围取得高度边界，不凭名称猜测。
- 玩家坐标只在玩家当前维度与目标维度一致时使用；其他维度回退原点，不虚构历史位置。
- 非有限数和超出整数范围的玩家坐标会被忽略，不会造成预览线程崩溃。

### 3. 32 格高度带采样

- 保留 V0.2 地表总览采样路径，主世界地表行为没有被替换。
- 高度带按每个 X/Z 柱从高到低寻找可通行地面，并允许跨 Section 检查上方两格。
- 缺失 Section 按空气处理；空高度带保持为空，不伪造方块。
- Mod 维度高度探测最多采纳 8 个可解析区块，并对单个损坏或不支持区块进行隔离，继续尝试同一 Region 和后续 Region。

### 4. 多维度生成与缓存

- `PreviewGenerator` 支持显式 `PreviewRequest`，同时保留 V0.2 主世界地表兼容入口。
- Region 目录必须位于对应世界目录内部，生成器和缓存均执行路径边界验证。
- 缓存渲染版本更新为 `0.2.1-layer-1`，缓存目录按世界和维度隔离，文件按图层区分。
- 元数据记录维度 ID、图层类型、最小/最大 Y、中心、Region 状态和生成统计。
- 0 字节 `.mca` 作为整合包创建的空 Region 占位文件处理为“缺失数据”；非零但头部损坏的文件仍报告失败。

### 5. JavaFX 维度与平滑高度选择

- 世界预览保留维度下拉框，并使用“地表总览”按钮、当前 Y/高度带标签和平滑高度滑块切换图层。
- 首次只生成当前维度的默认图层；用户切换维度、选择地表总览或释放高度滑块后，在后台按需生成或读取对应缓存。
- 每个维度在本次程序运行期间独立保存滑块坐标和当前图层；切换回该维度时恢复原值，不与其他维度共用。选择新世界或重新启动程序后不会保留这些状态。
- Mod 维度高度扫描、区块采样和缓存写入均在后台任务执行。
- 切换请求会取消旧任务，旧任务结果不会覆盖当前界面。
- 维度解析失败时清空并禁用旧图层控件，避免上一维度的状态错误残留。
- 导出按钮继续导出当前实际显示的 PNG，不改变 V0.2 的路径保护和已有文件保护。

## 开发中发现并修正的问题

1. 无 Mod 维度时，首版代码尝试对不可变空列表排序，会导致 `UnsupportedOperationException`；改为可变副本后排序。
2. 首版 Mod 维度扫描会进入 Region、实体和 POI 子树，真实整合包中产生不必要遍历；改为目录级发现并跳过数据子树。
3. 首版 Mod 高度探测只查看最前面的少量 Region 文件，Aether 样本无法取得高度；改为继续扫描后续 Region，直到获得足够有效区块。
4. 某个坏区块会让高度探测放弃整个 Region；改为逐区块隔离，并加入“坏区块在前、有效区块在后”的回归测试。
5. 非有限玩家坐标可能在取整时产生无意义中心；增加有限数和整数范围检查。
6. 自定义地表图层可携带非规范高度范围，可能产生不一致缓存身份；地表层固定为规范 `0..0`。
7. 缓存入口最初缺少维度 Region 路径边界校验；现在与生成器使用同一世界内部约束。
8. 切换到无法解析的维度后，图层下拉框可能重新启用上一维度的旧列表；开始新请求时立即清空图层状态。
9. 真实整合包中的 0 字节 `.mca` 占位文件被重复计为数千个失败区块；诊断确认全部为 `TRUNCATED_HEADER` 后，空文件改计缺失，三组真实预览的失败数均降为 0。
10. Gradle 8.7 提示测试平台启动器依赖将不再自动补入；显式加入 `junit-platform-launcher:1.10.2` 后，`--warning-mode all` 无弃用警告。

详细逐文件记录位于 `docs/issues/resolved/V0.2.1/`。
