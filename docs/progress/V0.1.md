# V0.1 存档管理器 开发进度记录

## 开发目标
完成最小可用产品（MVP）：扫描 Minecraft 存档目录并显示基础信息。

## 当前开发阶段
V0.1 存档管理器开发已完成，进入结项验收状态。

## 实际完成
- 搭建了 Gradle + Java 21 + JavaFX 项目骨架。
- 编写了基础配置文件 `build.gradle` 和 `settings.gradle`。
- 创建了 `App.java` 作为应用程序的入口。
- 引入了 `net.kyori:adventure-nbt` 作为 NBT 解析库，以及 `JUnit 5` 用于测试。
- 创建了 `WorldInfo` 数据传输对象 (DTO)。
- 实现了 `LevelDatReader` 核心解析逻辑，提取 `level.dat` 中的世界信息。
- 编写了 `LevelDatReaderTest` 以验证异常情况处理。
- 实现了 `WorldScanner` 类，能够扫描指定目录下的 Minecraft 存档。
- `WorldScanner` 内部实现了容错机制，跳过损坏存档并正常返回合法存档列表。
- 实现了默认游戏目录候选路径识别，以支持不同操作系统（Win/Mac/Linux）的默认 Minecraft 目录。
- 编写了 `WorldScannerTest` 以验证边界情况（如扫描不存在的路径）。
- 设计并创建了主界面的 FXML 布局文件 `main.fxml`，采用 `SplitPane` 实现左右分栏（左侧存档列表，右侧详情面板）。
- 编写了控制逻辑基类 `MainController.java`，完成了 FXML 节点（`ListView` 及各种详情 `Label`）的 `@FXML` 注入和初始化重置功能。
- 更新了 `App.java`，正式通过 `FXMLLoader` 加载 `main.fxml` 界面。
- 编写了 `WorldListCell` 自定义 UI 控件，支持在存档列表中同时显示 `icon.png` 和世界名称。
- 在 `MainController` 启动时集成了 `WorldScanner`，自动扫描本地默认存档目录并将数据填充至列表。
- 实现了事件监听机制，点击列表项时自动将版本、游戏模式、游玩时间、种子和玩家坐标展示到右侧面板。
- 完成了静态代码审核与走查，确认所有的类依赖、布局文件与资源路径绑定均准确无误。
- **(Feature Update)** 引入了混合扫描机制与自定义目录配置，允许用户在界面通过按钮手动选择存档文件夹（采用 Preferences 自动保存持久化），并在无配置时自动回退扫描原版存档及第三方启动器版本隔离存档。
