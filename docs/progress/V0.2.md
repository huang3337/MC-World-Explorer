## 当前阶段

## 版本目标

在不启动 Minecraft、不依赖外部程序且不修改存档的前提下，生成并显示主世界静态俯视缩略图。

已确认的主要产品目标：

- 中心点依次使用主世界个人重生点、世界出生点、`(0, 0)`。
- 范围为 `1024 x 1024` 方块，输出为 `512 x 512` PNG。
- 使用方块地图颜色和高度明暗表现地形。
- 在程序内提供“世界预览”视图，并支持导出 PNG。
- 缓存、日志和导出文件全部保存在程序根目录。
- 不生成下界或末地缩略图。

完整设计见 `docs/superpowers/specs/2026-07-14-v0.2-world-thumbnail-design.md`。

### 开发中发现并修正的问题

1. 最初的路径解析先向上查找项目标记，导致项目目录内部生成的 jpackage 镜像被误判为项目根。调整为优先识别主 JAR 所在的 `app/` 或 `lib/` 布局，并增加嵌套目录回归测试。
2. 第一版允许外部系统属性覆盖程序根目录，可能违反严格便携边界。生产初始化现已始终自行检测根目录，系统属性只用于把检测结果传给 Logback。
3. 新增运行时回退测试最初使用单层目录创建不存在的父路径，在进入被测代码前失败。修正测试夹具后完整测试通过。
4. 迁移旧日志时，Windows PowerShell 所用 .NET 不支持 `Path.GetRelativePath`，首次脚本在删除源文件前停止并产生一个已校验的重复副本。确认源文件未受影响后清理重复副本，改用受前缀检查保护的相对路径计算，最终迁移和哈希验证成功。
5. 最终 jpackage 验收结束时，启动器退出后 JVM 短暂持有 JAR 文件锁，第一次清理临时镜像失败。使用 `jps` 确认验证 JVM 已结束，等待文件锁释放后再次清理成功，现有 V0.1.1 软件包未受影响。

## 第三阶段：Region 文件头、区块定位和解压

## 第四阶段：现代 Section、Palette 和地表采样

### 实际完成

- 新增 `SurfaceSampler`、`ChunkSurface` 和 `SurfaceColumn`，从解压后的区块 NBT 中取得每个 `16 x 16` 方块列的最高非空气方块与 Y 坐标。
- 支持 1.18+ 根级 `sections/block_states/palette/data` 布局。
- 支持较早 Palette 时代的 `Level/Sections/Palette/BlockStates` 布局，不扩展到无 Palette 的旧区块结构。
- 根据 long 数组长度区分紧凑跨 long 和每个 long 独立填充两种位排列，支持非 2 的幂 Palette 大小。
- 识别 `minecraft:air`、`cave_air` 和 `void_air`，从最高 Section 向下寻找可见地表。
- 支持负 Section Y；不依赖固定世界高度。
- Palette、Section、位存储、NBT 和索引错误使用结构化原因报告，一个区块失败不会触发存档写入。
- 单区块 Palette 上限为 4096，NBT 读取继续受 64 MiB 上限保护。

### 测试与真实存档验收
