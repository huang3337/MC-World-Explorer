## 版本目标

在不启动 Minecraft、不依赖外部程序且不修改存档的前提下，生成并显示主世界静态俯视缩略图。

已确认的主要产品目标：

- 中心点依次使用主世界个人重生点、世界出生点、`(0, 0)`。
- 范围为 `1024 x 1024` 方块，输出为 `512 x 512` PNG。
- 使用方块地图颜色和高度明暗表现地形。
- 在程序内提供“世界预览”视图，并支持导出 PNG。
- 缓存、日志和导出文件全部保存在程序根目录。
- 不生成下界或末地缩略图。

完整设计见 `docs/superpowers/specs/2026-07-14-v0.2-world-thumbnail-design.md`。

## 第一阶段：严格便携路径和日志迁移

### 实际完成

- 将 Gradle 开发版本更新为 `0.2.0-SNAPSHOT`。
- 新增 `storage/PortablePaths.java`，统一解析开发环境、Gradle 分发目录和 jpackage 应用镜像的程序根目录。
- `Launcher` 在加载 `App` 和 Logback 前创建并验证程序根目录下的 `logs/`。
- 日志目录不可写时显示 JavaFX 错误对话框，提示用户把程序移动到普通可写目录。
- Logback 从 AppData 迁移到 `${mcworldexplorer.home}/logs`，不提供 AppData 静默回退。
- 应用启动日志记录实际程序根目录，便于定位存储位置。
- 外部 JVM 参数不能覆盖生产启动检测到的程序根目录。

### 测试与启动验收

- `gradlew.bat clean test`：9 个测试套件，共 32 个测试，30 个通过，2 个真实目录集成测试按设计跳过，0 个失败，0 个错误。
- 新增 6 个便携路径测试，覆盖显式测试根目录、开发类目录、jpackage `app/`、Gradle `lib/`、运行时回退和无效日志路径。
- 开发环境实际启动成功，日志记录程序根为项目根目录。
- 构建独立临时 jpackage 应用镜像，并从 `%TEMP%` 作为工作目录启动；进程保持响应，日志只出现在 EXE 所在目录的 `logs/`。
- jpackage 验收前后 AppData 旧日志的 SHA-256 和修改时间保持不变。
- 清理 AppData 旧日志后再次执行完整测试，`%APPDATA%\MC World Explorer` 没有重新生成。

### 开发中发现并修正的问题

1. 最初的路径解析先向上查找项目标记，导致项目目录内部生成的 jpackage 镜像被误判为项目根。调整为优先识别主 JAR 所在的 `app/` 或 `lib/` 布局，并增加嵌套目录回归测试。
2. 第一版允许外部系统属性覆盖程序根目录，可能违反严格便携边界。生产初始化现已始终自行检测根目录，系统属性只用于把检测结果传给 Logback。
3. 新增运行时回退测试最初使用单层目录创建不存在的父路径，在进入被测代码前失败。修正测试夹具后完整测试通过。
4. 迁移旧日志时，Windows PowerShell 所用 .NET 不支持 `Path.GetRelativePath`，首次脚本在删除源文件前停止并产生一个已校验的重复副本。确认源文件未受影响后清理重复副本，改用受前缀检查保护的相对路径计算，最终迁移和哈希验证成功。
5. 最终 jpackage 验收结束时，启动器退出后 JVM 短暂持有 JAR 文件锁，第一次清理临时镜像失败。使用 `jps` 确认验证 JVM 已结束，等待文件锁释放后再次清理成功，现有 V0.1.1 软件包未受影响。
