# 统一维度图层系统设计

## 状态与范围

本设计已由开发者于 2026-07-18 确认，并已在 V0.2.1 多维度预览与洞穴分层中完成核心能力的阶段性实施和自动化/真实存档验收；V0.2.1 版本开发尚未确认完成。

开发者后续确认 V0.2 不单独构建软件包，并明确授权进入 V0.2.1。拖动、缩放、坐标跳转和标记仍属于 V0.3，没有纳入本次实现。

## 目标

- 所有维度使用同一套图层模型，不为主世界、下界、末地和 Mod 维度分别维护互不兼容的渲染流程。
- 保留地表总览的直观入口，同时让玩家按高度查看洞穴、地下建筑、下界道路和多层结构。
- 图层按需生成，避免首次选中存档时同时读取并缓存整个垂直范围。
- 继续保持 Minecraft 存档只读和严格便携存储。

## 统一图层模型

每个可预览维度都提供两类图层。

### 地表总览

`SURFACE_OVERVIEW` 是自适应虚拟层。它在每个 X/Z 方块柱中寻找整个维度内最高的可见地表，用于快速理解开放地形。

主世界、末地和未知 Mod 维度默认先显示该层。对于存在封闭顶部的维度，该层可能显示顶部结构，但用户可以切换到普通高度带。

### 高度带

`HEIGHT_BAND` 将维度实际垂直范围按 32 格划分，例如 `Y 32-63`、`Y 64-95`。边界从维度定义或实际区块 Section 范围取得，不假定所有维度都采用主世界高度。

高度带不是单一 Y 平面的截面。在每个 X/Z 位置中，它寻找该范围内最高的可通行地面：地面方块为非空气，上方至少保留两格可通行空间。这样可以显示洞穴道路、地下基地和下界内部结构，而不是产生大面积实心岩层截面。

## 默认图层

- 玩家当前位于目标维度且坐标可用时，默认选择玩家 Y 所在的高度带。
- 主世界、末地和未知 Mod 维度在没有目标维度玩家坐标时默认使用地表总览。
- 下界在没有目标维度玩家坐标时默认使用包含 `Y=64` 的高度带，避免首先显示基岩顶部。
- 基岩顶部不使用独立渲染器；它属于对应高度带，未来界面可以提供“下界顶部”快捷入口。
- Minecraft 存档未可靠保存每个维度的历史最后位置时，程序不得虚构该位置。

## 生成与缓存

- 首次进入维度时只生成默认图层。
- 用户选择尚未缓存的高度带时再生成该层。
- 切换图层时复用现有 Region、区块解压和方块颜色能力，但采样结果按图层范围隔离。
- 缓存身份至少包含世界标识、维度 ID、中心坐标、图层类型、高度范围和渲染版本。
- 每个图层独立判断 Region、`level.dat` 和相关维度数据是否变化。
- 缓存继续保存在程序根目录 `cache/`，不得写入世界存档或 AppData。

建议缓存结构：

```text
cache/
└─ 世界标识/
   └─ minecraft_the_nether-维度哈希/
      ├─ surface-overview.png
      ├─ surface-overview.json
      ├─ y-32-63.png
      └─ y-32-63.json
```

## Mod 维度边界

统一图层模型适用于已经被程序发现的 Mod 维度。V0.2.1 结合真实整合包样本采用 `dimensions/<namespace>/<dimension path>/region` 目录发现规则，从可解析区块 Section 取得实际高度范围；无法确定范围时不猜测。缺少当前维度玩家坐标时使用维度原点作为中心。

未知方块继续使用稳定回退颜色。是否读取 Mod 资源包或纹理不属于本设计，也不能因统一图层系统而提前引入。

## 错误处理

- 某个图层无法读取时，只影响该图层，不应破坏其他图层缓存。
- 高度带没有可通行地面时显示为空图层状态，不伪造方块。
- 不支持的 Region 压缩或区块布局继续报告结构化原因。
- 维度范围异常、缓存目录不可写或生成被取消时，必须向界面返回明确状态。

## 实施验收结果

- 同一套图层接口可用于主世界、下界、末地和测试 Mod 维度。
- 主世界地表总览不能因加入高度带而退化。
- 下界内部高度带能避开顶部基岩对整张图的遮蔽。
- 切换到已有缓存图层不重新生成；未缓存图层按需生成。
- 真实存档验收前后相关文件的大小、修改时间和 SHA-256 保持一致。
- 不实现未经路线图确认的全维度预生成、三维体素浏览或 Mod 纹理加载。

2026-07-18 的阶段性验收共运行 27 个测试套件、105 个测试；启用当时的真实目录后全部通过。真实样本发现 8 个下界、7 个末地和 54 个 Mod 维度入口，并分别成功生成下界高度带、末地地表和 Aether 地表预览。相关 `level.dat` 与 Region 文件在生成前后的大小、修改时间和 SHA-256 完全一致。该段是带日期的历史快照，后续样本变化不反写本段数据。
