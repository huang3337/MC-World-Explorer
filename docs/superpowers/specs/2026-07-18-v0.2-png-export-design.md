# V0.2 PNG 导出设计

## 状态与范围

本设计已由开发者于 2026-07-18 确认，用于完成 V0.2 正式开发顺序中的第 8 阶段“PNG 导出和目录错误处理”。

本阶段只导出程序当前已经显示的世界预览 PNG，不重新生成地图，不改变缓存失效规则，也不加入 V0.3 的拖动、缩放、坐标跳转或标记功能。

## 目标

- 用户在程序内成功看到世界预览后，可以一键导出当前图片。
- 默认导出位置固定为程序根目录下的 `exports/`，保持严格便携。
- 导出过程不覆盖已有文件，不修改缓存源文件，更不写入 Minecraft 世界目录。
- 默认目录不可用时给出明确错误，并允许用户主动选择其他保存位置。
- 导出失败只影响本次导出，不影响当前预览和缓存。

## 用户界面

世界预览页新增“导出 PNG”按钮。

- 未选中可预览存档、正在生成、生成失败或图片加载失败时，按钮禁用。
- 缓存图片或新生成图片已经成功显示时，按钮启用。
- 用户切换存档或当前预览被清空时，按钮立即禁用，避免导出上一份存档的图片。
- 导出期间不重新读取 Region 文件，也不重新渲染；按钮只处理当前显示图片对应的缓存 PNG。
- 默认导出成功后，在预览页现有状态区域显示完整导出路径，不自动打开文件夹。

界面不新增尺寸、格式、质量等选项。V0.2 只导出原始 `512 x 512` PNG。

## 默认导出位置与命名

`PortablePaths` 增加程序根目录下的导出目录：

```text
MC World Explorer/
└─ exports/
```

`exports/` 在用户第一次执行导出时按需创建，程序启动时不提前创建。

默认文件名格式为：

```text
世界名称-20260718-153045.png
```

命名规则如下：

- 世界名称沿用 Windows 文件名安全清理规则，替换非法字符并移除尾随点和空格。
- 空白名称、`.`、`..` 或清理后为空的名称使用稳定的通用名称 `world`。
- 文件名需要限制到安全长度，为时间戳、重复编号和 `.png` 后缀保留空间。
- 时间戳使用本地时间，精确到秒，格式固定为 `yyyyMMdd-HHmmss`。
- 同一秒内出现重名时依次追加 `-2`、`-3`，绝不覆盖已有文件。

## 导出组件

新增独立的 `PreviewExporter`，只负责文件导出规则，不依赖 JavaFX。

它接收当前预览 PNG、世界名称、目标目录或目标文件，并完成：

1. 校验源 PNG 是存在的普通文件。
2. 生成安全且唯一的默认文件名。
3. 校验目标不位于当前 Minecraft 世界目录或其后代目录中。
4. 在目标目录中写入临时文件。
5. 优先使用原子移动完成最终落盘；文件系统不支持原子移动时，使用不覆盖目标的普通移动。
6. 返回最终导出文件的绝对路径。

导出采用文件复制，不重新编码图片，保证导出内容与当前缓存 PNG 字节一致。临时文件必须位于最终目标目录中，成功后删除临时状态；失败时尽力清理临时文件，并保留原缓存和已有导出文件。

`MainController` 只负责：

- 跟踪当前成功显示的预览图片路径和对应世界目录。
- 管理导出按钮启用状态。
- 调用 `PreviewExporter`。
- 显示成功路径或错误提示。
- 在默认目录不可用时发起备用位置选择。

文件名清理、唯一编号、原子落盘和路径安全判断不放入控制器。

## 默认导出流程

1. 用户选中存档，程序显示有效的缓存图片或新生成图片。
2. 用户点击“导出 PNG”。
3. 程序尝试创建或使用程序根目录下的 `exports/`。
4. `PreviewExporter` 生成唯一文件名并复制当前缓存 PNG。
5. 成功后，界面显示最终绝对路径，预览保持不变。

导出不会改变当前缓存元数据，也不会把导出文件纳入缓存复用判断。

## 默认目录失败与备用位置

如果程序根目录下的 `exports/` 无法创建或写入，程序不得静默改写到 AppData、用户目录、临时目录或 Minecraft 存档目录。

界面应明确说明默认导出目录不可用，并提供两个操作：

- 取消本次导出。
- 由用户通过 JavaFX `FileChooser` 明确选择另一个 PNG 保存位置。

备用保存遵循以下规则：

- 默认文件名仍使用同一套安全命名规则。
- 目标已存在时不得静默覆盖；应提示用户重新选择或取消。
- 目标位于当前世界目录或其任意后代目录时拒绝导出，并说明程序不会向 Minecraft 存档写入文件。
- 用户取消选择时不视为程序错误，当前预览和状态保持可用。

本阶段只在默认目录创建或写入失败时提供备用位置，不把每次导出都改为“另存为”。

## 状态与并发安全

- 导出操作绑定点击时的预览源路径和世界目录快照。
- 切换存档会清空控制器中的可导出状态并禁用按钮。
- 如果点击后源缓存被删除、替换为目录或无法读取，本次导出失败并显示明确信息。
- 导出完成后不因当前选择已变化而覆盖新存档的预览状态；仅在导出仍对应当前预览请求时更新成功提示。
- 导出只复制一张约 `512 x 512` 的 PNG，不启动新的 Region 后台任务。

## 错误处理

至少区分以下错误：

- 当前没有可导出的预览。
- 源 PNG 不存在、不是普通文件或无法读取。
- 默认 `exports/` 无法创建或写入。
- 目标文件已存在。
- 目标位于 Minecraft 世界目录中。
- 临时文件写入、复制或最终移动失败。

面向用户的提示保持简洁并说明可采取的操作；详细异常写入程序根目录 `logs/`。任何失败都不得删除或修改缓存源 PNG、已有导出文件和存档文件。

## 测试与验收

### 自动化测试

- 世界名称含 Windows 非法字符、尾随点或空格、空名称、`.`、`..` 和超长名称时，生成安全文件名。
- 时间戳格式固定，同一秒重名时生成 `-2`、`-3`，不覆盖已有文件。
- 首次导出时创建 `exports/`，成功文件与源 PNG 字节完全一致。
- 写入过程使用目标目录临时文件，成功后不存在遗留临时文件。
- 源 PNG 缺失、不可读或不是普通文件时明确失败。
- 默认目录不可创建或不可写时明确失败，不回退到 AppData。
- 备用目标在当前世界目录或其后代目录时拒绝导出。
- 备用目标已存在时不覆盖。
- 切换存档或清空预览后导出按钮禁用，不保留旧图片导出状态。
- 缓存加载成功和新图生成成功两条路径均能启用导出。

### 整体验收

- 默认完整 Gradle 测试通过。
- 显式真实存档测试通过，相关 `level.dat` 和 Region 文件在测试前后大小、修改时间和 SHA-256 保持一致。
- 实际启动程序，分别从缓存命中和新图生成结果执行导出。
- 检查默认导出文件位于程序根目录 `exports/`，能正常打开且与当前预览一致。
- 检查重名保护、默认目录失败提示和备用位置选择。
- 使用 `git status`、`git check-ignore` 和 `git ls-files` 确认 `exports/` 仍被忽略，不进入仓库。

## 明确不包含

- 自动打开导出目录。
- 批量导出多个世界。
- JPEG、WebP、SVG 或其他格式。
- 自定义导出尺寸、裁剪、缩放或重新渲染。
- 每次点击都弹出保存位置选择器。
- 把导出路径保存为用户偏好设置。
- V0.3 的地图拖动、连续缩放、坐标定位和标记系统。
- 下界、末地和 Mod 维度导出。

## 完成标准

用户可以对当前成功显示的主世界预览执行一次点击导出。默认文件安全、唯一且位于程序根目录 `exports/`；默认目录失败时只有用户明确选择后才写入其他位置；任何流程均不修改 Minecraft 存档、缓存源文件或已有导出文件。
