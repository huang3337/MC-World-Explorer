# MC World Explorer V0.2 世界缩略图设计规范

## 文档状态

- 设计状态：已确认进入 V0.2
- 适用版本：V0.2
- 整理日期：2026-07-14
- 实现状态：严格便携路径和日志迁移、缩略图中心点解析已完成

本文档把开发者已经确认的产品方向整理为正式设计。V0.2 已于 2026-07-14 开始，当前完成严格便携路径和日志迁移、缩略图中心点解析；Region 解析、图片生成和预览界面仍按阶段逐项实施。

## 目标

在不启动 Minecraft、不依赖外部程序且不修改世界存档的情况下，根据 Java 版世界的 Region 数据生成主世界静态俯视缩略图，并在 MC World Explorer 中直接查看和导出。

V0.2 的完成结果是“能够生成并显示固定范围的世界俯视图”，不是完整地图浏览器。拖动、连续缩放、坐标跳转和标记系统属于 V0.3。

## 已确认的产品范围

### 维度与中心点

- 只生成主世界缩略图，不生成下界或末地缩略图。
- 优先使用玩家在主世界设置的个人重生点，通常对应床的位置。
- 个人重生点缺失、无效或不属于主世界时，使用世界出生点。
- 两类坐标都无法取得时，最终回退到主世界 `(0, 0)`。

### 范围与输出

- 读取中心周围 `1024 x 1024` 方块范围。
- 生成 `512 x 512` PNG，每个像素表示 `2 x 2` 方块范围的汇总结果。
- 使用地表方块颜色与高度明暗表现地形，不加载 Minecraft 客户端、资源包或外部渲染程序。
- V0.2 使用固定分辨率位图。交互式瓦片、无级缩放和拖动浏览留到 V0.3。

### 程序内查看

- 右侧详情区域增加“存档信息”和“世界预览”两个视图。
- 用户选择存档后，程序在后台检查缓存；缓存有效时直接显示，否则后台生成。
- 生成过程不得阻塞 JavaFX 界面。
- 用户切换存档时，旧任务不得把结果显示到新选择的存档上。
- 预览区域明确显示未生成、生成中、成功、部分数据缺失和生成失败状态。

### 缓存与导出

- 缓存保存到程序根目录的 `cache/世界标识/`。
- 每个世界至少保存 `world_preview.png` 和 `preview-metadata.json`。
- 缓存元数据记录渲染版本、中心坐标、范围、输出尺寸以及相关 `level.dat` 和 Region 文件状态，用于判断缓存是否失效。
- 用户主动导出的 PNG 默认保存到程序根目录的 `exports/`，文件名包含可读世界名称和导出时间。
- 文件名冲突时生成新名称，不覆盖已有导出文件。
- 程序目录不可写时明确提示；导出操作可以让用户主动选择其他目录，但程序不得静默回退到 AppData。

## 严格便携存储边界

本设计遵循 `docs/decisions/DECISION-001.md`：

- 开发环境将项目根目录视为程序根目录。
- Windows 便携版将 EXE 所在目录视为程序根目录。
- 日志写入 `logs/`，缩略图缓存写入 `cache/`，导出写入 `exports/`，未来配置写入 `config/`。
- 不使用 `%APPDATA%` 或 `%LOCALAPPDATA%` 保存应用生成数据。
- 不在 Minecraft 世界目录中写入预览图、缓存、日志或配置。
- `cache/`、`logs/`、`exports/` 和 `config/` 不进入 Git。

正式实现缩略图前，必须先完成统一程序根目录解析和日志路径迁移，并通过开发环境与便携版实际启动验证。

## 建议模块边界

### PortablePaths

负责解析程序根目录以及 `cache/`、`logs/`、`exports/` 路径，检查目录可写性。它不负责 Region 解析或界面提示。

### PreviewCenterResolver

只负责按照“主世界个人重生点、世界出生点、`(0, 0)`”的顺序确定预览中心，并返回选择来源。

### RegionFileReader

只读打开主世界 `region/*.mca`，解析 Region 文件头、区块位置、时间戳和压缩类型，并输出解压后的区块 NBT 字节。它不解释 Section 或 Palette；损坏区块的失败不能导致程序修改或删除原文件。

### SurfaceSampler

从区块 Section、方块 Palette 和高度信息中取得地表方块，向渲染器提供与 JavaFX 无关的采样结果。

### PreviewRenderer

把地表采样转换为固定的地图颜色与高度明暗，输出 `512 x 512` PNG。未知 Mod 方块使用明确的回退颜色，不把“未知”伪装成已准确识别。

### PreviewCache

负责世界标识、缓存元数据、有效性检查和原子替换缓存文件。缓存失败不能影响世界存档，也不能阻止用户继续查看基础存档信息。

### PreviewService

编排中心点、Region 读取、渲染和缓存，但不直接操作 JavaFX 控件。

### 预览界面控制

负责后台任务、状态展示、选择切换保护和导出命令。解析和渲染逻辑不得放入控制器。

## 数据流程

1. 用户选择一个已识别的世界。
2. 程序解析预览中心和主世界 Region 目录。
3. 程序检查该世界的缓存元数据。
4. 缓存有效时读取 PNG；缓存缺失或失效时启动后台生成任务。
5. RegionFileReader 只读取覆盖目标范围的 Region 文件和区块。
6. SurfaceSampler 取得地表结果，PreviewRenderer 生成 PNG。
7. PreviewCache 将 PNG 和元数据写入程序根目录缓存。
8. JavaFX 线程只负责更新预览状态和显示最终图片。
9. 用户点击导出时，将当前有效 PNG 写入 `exports/` 或用户明确选择的位置。

## 错误与降级策略

- 世界没有 `region/`：显示“没有可用的主世界 Region 数据”，不影响基础信息查看。
- 个别 Region 或区块损坏：记录路径和原因，跳过损坏数据；仍有有效数据时生成部分缩略图并明确标记。
- 压缩类型或区块格式不支持：拒绝猜测解析，记录并显示兼容性提示。
- 方块名称未知：使用统一回退颜色并统计未知方块，不因单个 Mod 方块终止生成。
- 缓存目录不可写：显示明确错误，不写入 AppData，不向存档目录回退。
- 导出失败：保留现有缓存和预览，不覆盖用户已有文件。
- 内存或读取额度超限：终止当前生成任务并报告，不允许无限制读取损坏数据。

## 兼容性起始范围

开发者已于 2026-07-14 确认 `docs/decisions/DECISION-002.md`。V0.2 首轮实现标准 Anvil `.mca`、GZIP、ZLIB 和未压缩区块，并以独立的 Region 容器、压缩和区块 NBT 层为后续扩展保留边界。

LZ4、外部区块、旧 `.mcr` 和特殊 Mod 自定义格式首轮只检测并明确提示不支持。扩大实现范围或新增第三方压缩依赖前必须更新决策，不能在实现过程中无边界追加格式分支。

## 分阶段实施顺序

1. 统一便携路径并迁移日志。
2. 解析和测试预览中心点。
3. 解析和测试 Region 文件头、区块定位与解压。
4. 解析和测试现代区块 Section、Palette 与地表采样。
5. 实现和测试颜色映射、高度明暗与 PNG 输出。
6. 实现和测试缓存元数据与失效判断。
7. 接入 JavaFX 后台任务和内置预览。
8. 实现导出、重名处理和不可写目录提示。
9. 使用合成数据、损坏样本和真实存档执行只读验收。
10. 整体代码审查、文档同步和 Windows x64 便携包验收。

每个阶段只解决一个明确目标，完成测试和复核后再进入下一阶段。重大兼容性、依赖或架构变化必须暂停并取得开发者确认。

## 测试策略

- 路径测试：开发目录、便携目录、不可写目录和禁止 AppData 回退。
- 中心点测试：有效主世界重生点、其他维度重生点、缺失重生点和最终回退。
- Region 测试：文件头、边界区块、外部区块、压缩类型、截断和损坏输入。
- 采样测试：负坐标、不同世界高度、空 Section、未知方块和跨 Region 范围。
- 渲染测试：固定尺寸、确定性输出、颜色回退和高度阴影。
- 缓存测试：命中、失效、损坏元数据、写入失败和文件名冲突。
- UI 测试：后台执行、快速切换存档、失败状态和成功显示。
- 真实存档验收：生成前后核对所读取存档文件的大小、修改时间和 SHA-256，证明没有写入。
- 便携包验收：确认应用生成数据只出现在程序目录，AppData 中不产生 MC World Explorer 数据。

## 完成标准

- 能围绕约定中心生成并显示主世界 `1024 x 1024` 范围的 `512 x 512` PNG。
- 缓存有效时可复用，相关世界数据变化后能够重新生成。
- 原版和目标整合包存档中的现代 Anvil 数据能够被安全处理；不支持的数据有明确提示。
- 生成期间界面保持响应，切换存档不会显示错配图片。
- 导出功能不会覆盖已有文件，并正确处理不可写目录。
- 不启动 Minecraft，不依赖外部渲染程序，不向世界存档写入数据。
- 开发环境和 Windows x64 便携版均不再向 AppData 写入应用数据。
- 自动化测试、真实目录只读验收、整体代码审查和便携版实际启动验收全部通过。

## V0.2 明确不开发

- 下界和末地缩略图。
- 地图拖动、连续缩放、瓦片地图和坐标跳转。
- 玩家、出生点、传送门或建筑标记。
- Mod 资源包和方块纹理加载。
- 三维浏览、自动巡航和动态壁纸。
- 向 Minecraft 存档写入封面、索引、缓存或配置。

上述内容不得因为实现方便或测试需要被提前加入 V0.2。
