# 世界预览中央状态文字修复设计

## 问题

缓存缩略图成功加载后，图片中央仍显示“已加载缓存 · 中心 X, Z”。同一状态已经显示在预览上方，中央重复文字遮挡地形并影响观感。

根因是缓存命中路径先通过 `showPreviewImage()` 隐藏中央占位文字，随后又调用 `setPreviewIdle()`，把占位文字重新设为可见。

## 已确认方案

采用方案 A：保留上方状态文字，中央标签只作为没有可显示图片时的占位和反馈。

中央标签显示场景：

- 尚未选择存档。
- 正在生成缩略图。
- 存档无法生成缩略图。
- 缩略图生成或读取失败。

中央标签隐藏场景：

- 成功加载有效缓存图片。
- 成功生成并加载新图片。

## 实现范围

- 缓存命中后只结束进度状态并更新上方状态文字，不再调用会显示中央占位文字的空闲状态方法。
- 提取可单元测试的成功显示状态判断，覆盖缓存加载和新图生成两条路径。
- 不删除中央标签，不修改预览布局，不改变缓存、坐标、图片生成或存档读取逻辑。

## 验证

- 新增回归测试，确认存在有效图片时中央占位文字应隐藏。
- 运行 `MainControllerTest` 和完整 Gradle 测试。
- 启动程序并加载已有缓存，确认上方状态保留且图片中央无重复文字。

## 边界

本修复属于 V0.2 内置预览 UI 健壮性修正。不会实现 PNG 导出、拖动、缩放、坐标跳转或其他 V0.3 功能。
